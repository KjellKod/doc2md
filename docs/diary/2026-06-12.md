## "Corrupt" Mac build artifact on run 27436746323 (2.8.8): diagnosis, no defect

### Context
Kjell reported the Mac build broken: the artifact downloaded from the Actions
run page for 2.8.8 (run 27436746323) "says it's corrupt," and there was no DMG
to download, only the app. He suspected an interrupted quest (the old Mac P12
signing work) had left the fix unlanded.

### Human Inputs to Remember
- Analysis was requested, not a fix. Reported findings and stopped.
- Kjell tests releases by downloading from the GitHub UI; the Actions run page
  was his natural first stop, not the Releases page.

### Findings (no code defect)
- Run 27436746323 succeeded end to end; both jobs green.
- The only run-page artifact, `doc2md-build` (~3.1 MB), is the UNSIGNED
  intermediate handed from the build job to the sign/publish job:
  `doc2md-app.tar.gz` + Sparkle `sign_update` + `release-meta.json`,
  7-day retention. Launching that unsigned, quarantined app produces
  macOS Gatekeeper's "damaged and can't be opened" message, which reads
  as "corrupt." That is expected for the unsigned intermediate.
- The real deliverables live on the Release, not the run page:
  https://github.com/KjellKod/doc2md/releases/tag/2.8.8 has
  `doc2md-2.8.8.dmg`, `doc2md-2.8.8.zip`, and `appcast.xml`.
- Verified the DMG locally: `hdiutil verify` checksum VALID; `spctl --assess`
  accepted, `source=Notarized Developer ID`.
- The remembered P12 quest was NOT lost or interrupted. It landed as
  PR #160 (strip whitespace before base64-decoding the P12), #163
  (`-f pkcs12` on `security import`), and #164 (default keychain + search
  list). Diary 2026-05-29 has the full story; the `save/mac-p12-decode-741e4a7`
  rescue branch served its purpose and is gone.
