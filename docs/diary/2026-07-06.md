### Changes Made
- No source or doc changes, analysis only. Findings recorded here and in the
  conversation summary for the joint sharpen.
- Key grounded facts collected:
  - Reminder cadence is implemented and session scoped:
    `apps/macos/doc2md/Licensing/LicenseReminderController.swift:32`
    (save 10, then every 25 saves, counter resets each launch).
  - Verifier is pure offline with a 5 minute clock skew allowance and a hard
    `expired` result: `LicenseVerifier.swift:45,81-87`. No grace state, no
    network calls exist today.
  - Issuer spec explicitly rejects online entitlement validation, activation
    seats, and revocation polling for the public app, while allowing future
    non-blocking online calls via explicit decision (spec lines 150 to 176).
  - Decision record is Accepted with Lemon Squeezy preferred, Paddle fallback,
    and "no fixed trial period" as a binding evaluation constraint.

### Findings (for the joint sharpen)
1. Resolved (my position): reminder cadence IS the trial; a time boxed trial
   would contradict the accepted decision record. Keep shipped cadence.
2. Resolved (my position): clock rollback hardening stays out of scope, the
   spec's honest-user stance covers it.
3. Open, biggest one: Polar's license-key API is online activation/validation,
   which the issuer spec currently forbids in the app. Using Polar as interim
   issuer requires a narrow spec amendment (one-time activation at license
   entry, cached offline with grace, degrade to reminders). My recommendation:
   amend and postpone the custom Ed25519 issuer entirely for v1.
4. Open: MoR amendment Lemon Squeezy to Polar primary; keep Paddle fallback,
   do not fall back onto Stripe Managed Payments while it is preview.
5. Open: pricing, recommend single SKU $20/yr for v1, perpetual-until-next-
   major deferred (token contract already supports it later).
6. Open: offline grace duration if Polar validation is adopted, recommend 30
   days after last successful check, then expired-reminder state, never a cliff.
7. Open: web app copy, recommend strict silence in the hosted app UI even
   after go-live; GitHub README and doc2md.dev may mention the Mac app.
