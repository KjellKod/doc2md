## Quest: view-anchor-mode-switch_2026-05-09__1206

Outcome: completed on branch `followup/desktop-doc-ux-polish` (no

### Context

User reported that switching between Edit and Preview (and LinkedIn)
in the desktop app does not preserve the same logical view. The
previous implementation cached a `scrollTop / (scrollHeight -
clientHeight)` ratio per mode, which is structurally wrong because
the editor textarea and the rendered preview have very different
vertical scales (h1/h2 take 3-4x body line-height, code blocks scroll
horizontally, blockquotes bloom), and `pageMaxWidth` resizes mutate
each surface's `scrollHeight` independently.

### Human Inputs to Remember

- Implementation lands on the existing followup branch, not a
  `quest/...` branch. `branch_mode: "none"`.
- Anchor what is on top of the source view; same content should be
  on top of the destination view.
- Resize between switches must hold the anchor.
- Test with both Vitest (unit) and Playwright (hosted-web) where
  feasible.
- User was doing parallel manual edits to docs, `.github/`,
  `scripts/security_ci_guard.py`, and the `actions-node24-refresh`
  quest folder. Builder was constrained to source/test paths only.
