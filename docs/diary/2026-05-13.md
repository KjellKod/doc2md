## Changes Made

PR #123, six commits, all green CI:

- **Bug 2 fix (rendered DOM leak):** Deleted `applyRenderedFindHighlight`, `clearRenderedFindHighlight`, `findTextPosition`, `textNodesFor`, `removeEmptyRenderedInlineElements`, the `clearRenderedFindHighlight` call in `switchMode`, and the now-unused setActiveSelection branch in `useFindReplace.ts`. Added `src/components/findHighlightRehype.ts` (rehype plugin walking the hast tree by rendered-text offsets, wrapping matches in `<mark>`, handling cross-emphasis, zero-width regex matches with U+200B sentinel, and HTML entities). Added U+200B scrub in the `renderedViewText` snapshot effect so the sentinel doesn't shift offsets for subsequent queries (reviewer B finding).
- **Bug 1 fix (wrong-line scroll):** Deleted `scrollTextareaToMatch`. Extended `scrollTextareaToLine` with `offsetFraction` (0 = top, 0.5 = center, clamped). The find-active-match layout effect now calls `textarea.setSelectionRange(end, end)` then `scrollTextareaToLine(..., 0.5)`. New centering layout effect uses `element.querySelector("mark.markdown-rendered-find-highlight") + centerElementInScrollContainer` for preview/LinkedIn.
- **Cubic LinkedIn-mode regression:** The first iteration of the rehype-plugin fix only ran through `react-markdown`, so LinkedIn-mode lost active-match highlighting. Added `renderLinkedInSegmentChildren` helper that walks a cursor through `linkedinPreview`, intersects each segment with `[match.start, match.end)`, and wraps the matched portion in `<mark className="markdown-rendered-find-highlight">` inline. Playwright regression in `tests/e2e/find-linkedin-highlight.spec.ts`, load-bearing.
- **macOS classic-scrollbar wrap divergence:** Added `scrollbar-gutter: stable` to both `.markdown-edit-area` and `.markdown-find-overlay` so they reserve identical gutter regardless of OS scrollbar mode. Bug is not reproducible in headless Chromium (Playwright always uses overlay scrollbars even when targeting `::-webkit-scrollbar`), so the test in `tests/e2e/find-edit-overlay-wrap.spec.ts` pins the computed `scrollbarGutter` property on both elements; load-bearing on the CSS rule.
- **Dark-mode highlight legibility:** Edit overlay yellow dropped from 0.82 to 0.32 alpha + 0.85-alpha outline so dark-mode text underneath stays legible. Rendered `<mark>` (preview + LinkedIn) now forces `color: #1a1a1a` so light dark-mode text doesn't render light-on-yellow.

Tests: 593 unit, 7 find-related Playwright tests (find-highlight-leak symptom A/B + Enter variant, find-match-scroll, find-linkedin-highlight, find-edit-overlay-wrap × 2). All green on CI.
