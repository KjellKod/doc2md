## Bugs surfaced by manual Mac validation (must-fix in follow-up)

1. **Vertical drag inversion.** Formula `nextWidth = startWidth - (event.clientX - startX)` is correct for a right-positioned sidebar. Our sidebar is on the LEFT. Flip the sign. `src/App.tsx:284`, `src/desktop/DesktopApp.tsx:432`.
2. **T-shape artifact.** `.workspace-split-bar` is a workspace grid child stretched to row height. The preview panel shrinks inline via `editShellHeight`, but the workspace row stays tall, leaving the vertical bar dangling below the preview's bottom edge. Move `.workspace-split-bar` inside `.preview-panel` (or absolute-position it so it tracks preview height).
3. **Snap-to-MAX on first horizontal click.** `handleHeightResizeStart` calls `setEditShellHeight(measuredHeight)` on mousedown. For a long document, the natural panel height exceeds `MAX_EDIT_SHELL_HEIGHT = 2400` and is clamped on the spot, causing a visible jump. Remove the premature `setEditShellHeight` from mousedown; let mousemove handle it.

## UX research outcome (saved at .ws/ux/UX-Guidance.md, gitignored)

- The `ui-ux-pro-max` skill is a 99-rule searchable catalog. Useful for focus-ring tokens, color tokens, motion gating, typography. Light on desktop split-pane / IDE patterns; not a design system.
- **Preview-pane scroll containment:** recommend binding `.preview-panel` to viewport via `max-height: calc(100dvh - chrome)` so the inner textarea owns long-document scroll. Matches VS Code / Obsidian / Zed / JetBrains. Removes the current behavior where a long doc pushes the resize handle and the About section below the fold.
- **2400px clamp:** inherited from the old corner-handle implementation, not a UX decision. Replace with a viewport-relative ceiling recomputed on `resize`. Floor stays at `MIN_EDIT_SHELL_HEIGHT = 240`.
- **Three roadmap items:** unify focus-ring tokens across textarea + both handles + collapse rail; add discoverable double-click reset hints (`title`, `aria-description`); gate the workspace grid transition on `prefers-reduced-motion: no-preference`.
