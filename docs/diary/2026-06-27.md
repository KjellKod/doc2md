## Interactive task checkboxes inside table cells — branch `feat/table-cell-task-checkboxes`

### Context
Quest `table-cell-checkboxes_2026-06-27__0922`. Implemented the human-reviewed
design in `.ws/plan-table-cell-task-checkboxes.md` v3: a GFM table cell whose
leading marker is `- [ ]` / `- [x]` / `[ ]` / `[X]` now renders a real,
clickable checkbox in Preview and toggles the exact source marker, while Edit
mode keeps the literal markdown. Export shows the same checkbox, static.

### Human Inputs to Remember
- PR #191's paste-time glyph approach (`☐`/`☑` rewritten into source) was
  scrapped and must NOT be reused: it destroyed editable source and produced
  inert glyphs. `pasteToMarkdown.ts` stayed out of scope; `tableCheckbox.ts` was
  not recreated.
- Convertibility is decided on RAW SOURCE, never decoded HAST, so escaped
  `\[ \]` stays literal.
- `replaceTaskMarkerAtSourceLine` kept byte-for-byte unchanged; the table path
  is a SEPARATE `replaceTaskMarkerByIndex`, dispatched by the
  `data-task-marker-index` attribute. Ordered markers `1) [ ]` / `1. [ ]` must
  keep routing through the unchanged list path.
- Another developer was editing `.skills/` GPT/codex files on the same branch;
  stayed strictly within the feature's source/test/style/diary files.

### Changes Made
- New canonical module `src/components/preview/tableCellCheckbox.ts`: pure
  `recognizeCellMarker` / `enumerateRowMarkers` / `toggleRowMarkerByIndex`
  operating only on raw source. Single data contract for both render-side index
  assignment and write-back, so indices cannot drift. Out-of-range markerIndex
  is a no-op (fail-safe).
- New shared rehype plugin `src/render/tableTaskCheckboxRehype.ts`: synthesizes
  `<input type="checkbox" disabled>` for real markers, runs the raw-cell-count
  vs DOM-cell-count fail-safe (skips the whole row on mismatch, e.g. a pipe
  inside inline code), preserves trailing labels. Wired into BOTH
  `markdownToHtml.ts` (export) and `PreviewMode.tsx` (preview) → parity by
  construction. The two resolvers are intentionally different: export reads the
  normalized line; preview maps the formatted line back to the original source
  line via the existing line map.
- `sourceLineRehype.ts`: preview-only stamping of synthesized table inputs with
  `data-task-source-line`, `data-task-marker-index`, and an `aria-label` that
  uses the trailing label when present, else a line+index form so bare-marker
  cells in one row stay individually targetable.
- `PreviewMode.tsx`: input renderer parses `data-task-marker-index` defensively
  (NaN → legacy list path) and dispatches table inputs to
  `replaceTaskMarkerByIndex`.
- `taskCheckboxSource.ts`: added `replaceTaskMarkerByIndex` alongside the
  unchanged list toggler.
- CSS parity: table-cell checkbox selectors in `global.css` and
  `htmlExportStyles.ts`, sharing task-list size/alignment tokens, omitting the
  list negative left margin (no bullet to offset).

### Fix iteration 1 — large-document fallback parity
- Review-arbiter kept one low-severity finding
  (`arb-large-doc-table-checkbox-doc-gap-1`): the large-document virtualized
  fallback preview (`LargeMarkdownPreviewMode`) renders its table manually, not
  via ReactMarkdown, so `tableTaskCheckboxRehype`/`sourceLineRehype` never ran
  there and table-cell checkboxes stayed literal in that mode. The human elected
  FULL PARITY over the cheaper doc-note option.
- Fix: wired interactive synthesis directly into the fallback path
  (`src/components/preview/PreviewMode.tsx`). Reused the canonical
  `enumerateRowMarkers` on each row's ORIGINAL raw source line (the exact string
  write-back edits — `row.sourceLine` already indexes the original markdown, so
  no formatted→original mapping is needed and the source line is always
  accurate). Write-back routes through the same `replaceTaskMarkerByIndex` as the
  rich path; `replaceTaskMarkerAtSourceLine` stays byte-for-byte untouched.
- Safety note: the fallback's fail-safe is weaker than the rich path's (no
  remark-gfm `<td>` count to compare against), but write-back re-runs the same
  enumerator on the same raw line, so the toggled span is always a genuine
  marker offset — no wrong-byte mutation. Documented inline.
- Tests: added PreviewPanel workflow tests proving the fallback now renders
  table-cell checkboxes, toggles row 0 with a byte-correct write-back, and keeps
  escaped `\[ \]` literal. The previously-undocumented gap is now closed, so no
  "unsupported in fallback" caveat is needed.
