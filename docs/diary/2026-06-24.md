# 2026-06-24

## Mobile paste backslash-escape bug — branch `claude/doc2md-formatting-backslash-bug-ho61zw`

### Context
Kjell reported from mobile web: clicked "New", pasted an already-markdown bullet
list, and the formatting got corrupted with backslashes — `- **Tests.**` became
`\- \*\*Tests.\*\*`. Asked to analyze the reason and check whether tests already
cover it. "This is a big problem."

### Human Inputs to Remember
- Repro path: mobile web browser, New → paste markdown source text.
- Symptom signature: literal markdown punctuation escaped with backslashes plus
  trailing hard-break spaces — classic Turndown output over text nodes.

### Changes Made
- Root cause: `inlineStyleHasMeaningfulFormatting` in
  `src/components/preview/pasteToMarkdown.ts` treated benign inline styles as
  "meaningful" rich formatting, defeating the `htmlIsTrivialPasteWrapper` guard.
  Mobile clipboard HTML wraps each line in `<span>` with `font-weight:400`,
  `color`, `font-size` — none representable in markdown. Guard missed → paste
  routed through Turndown → every markdown char in text nodes escaped.
  - Two defects: (1) a `font-weight` fallthrough flagged any non-`"normal"`
    weight (incl. numeric `400`, which is normal) as emphasis; (2) `color`,
    `background`, `font-size` were in `MEANINGFUL_INLINE_STYLE_KEYS` despite
    producing no markdown.
- Fix: narrowed `MEANINGFUL_INLINE_STYLE_KEYS` to `font-weight`, `font-style`,
  `text-decoration`; removed the font-weight fallthrough. Bold (weight >= 600 or
  `bold`/`bolder`), italic, and strikethrough/underline still keep the HTML path.
- Verified empirically with a throwaway repro before/after (deleted): styled-span
  wrapper went `source: html, escaped: true` → `source: plainText, escaped: false`.
- Tests: existing coverage only used no-style or `font-family` wrappers (the
  latter not in the meaningful set), so this shape was uncovered. Added 4
  regression tests in `pasteToMarkdown.test.ts` — styled-span mobile wrapper,
  sub-threshold numeric weight, and a guard that real `font-weight:700` still
  routes through Turndown to `**bold**`.
- Full suite green: 78 files / 857 tests. Typecheck + eslint clean on touched
  files.

### Sources Consulted

- `src/components/preview/pasteToMarkdown.test.ts` (existing wrapper coverage).

### Security Notes
- No network calls, no secrets, no data-loss surface. Pure conversion-heuristic
  change; the conservative direction is "prefer the user's literal plain text."

### Open Questions
- Secondary path observed in repro (not fixed): when clipboard HTML carries
  smart quotes the plain text lacks (or vice versa), `normalizedHtmlText !==
  normalizedPlainText` defeats the wrapper guard and Turndown escapes. Left
  alone deliberately — normalizing quotes/dashes in the equality check would
  break the intended `—` vs `---` dash-substitution tests. Worth a measured
  follow-up if a real report surfaces.
