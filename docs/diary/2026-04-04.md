# 2026-04-04 — API Tooling Quest

## Context
Quest `api-tooling_2026-04-04__0801` completed in workflow mode. The work started as a feasibility pass for server and CLI reuse, then pivoted into a real build quest for a portable `@doc2md/core` package.

## Human Inputs to Remember
- User wanted an easy API or function call with an easy-to-install bundle or package, not another browser-tied library.
- User explicitly wanted batch input (`1..N`) with a configurable limit, unsupported files skipped, and markdown written to files rather than embedded in result JSON.
- User named concrete downstream consumers early: a resume screener in `~/ws/extra/candid_talent_edge`, an MCP-first PDF extraction tool, and a general document-to-markdown skill.
- User pushed hard on plan honesty and asked for the quest brief and plan to be updated to reflect the pivot from analysis into build scope.
- User explicitly required this to stay a full-blown quest, not a solo quest.

## Corrections Received
- The user and reviewers forced the plan to close several vague contracts before build: batch-limit behavior, filename collisions, packaging/installability, PDF parity, Node-vs-browser test isolation, and browser regression gates.
- Code review caught that concurrent duplicate-basename writes were race-prone.
- Code review caught that per-document I/O failures were aborting the whole batch.
- Code review caught that CLI numeric arguments could bypass limits or degrade execution into malformed results.

## Decisions Made
- Keep the package ESM-only, file-path based, and metadata-first: markdown is written to disk and JSON reports status, paths, and quality.
- Reuse shared converters from `src/converters/` rather than fork them into a Node-only pipeline.
- Enable Node compatibility through explicit runtime/provider setup instead of `typeof window` branching scattered across converter files.
- Validate installability with an `npm pack` clean-consumer smoke test, not just in-repo execution.
- Treat fix-loop findings as first-class contract work, not cosmetic cleanup.

## Changes Made
- Added the `packages/core/` workspace package with batch and single-document APIs, CLI entrypoint, Node compat bootstrapping, and focused package tests.
- Updated shared converter internals so text, binary, rich text, PPTX, and PDF conversion can run in Node without breaking the existing browser app.
- Added a browser-generated PDF golden artifact and Node-side parity test.
- Hardened output collision handling, per-document error rows, and CLI numeric parsing after review.
- Wrote the quest journal and Dexter memoir for this quest.
