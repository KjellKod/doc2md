### Security Notes
- Forbidden-API grep targets `apps/macos/doc2md/ShellBridge.swift` only. `AppScheme.swift` legitimately uses `FileManager` for the `doc2md://` scheme handler; keeping the grep scope narrow preserves that. Pattern: `\b(FileManager|NSOpenPanel|NSSavePanel|FileHandle|replaceItem|replacingItem|startAccessingSecurityScopedResource|stopAccessingSecurityScopedResource|securityScopedResource)\b|\.write\(to:`.
- Scripts use `set -euo pipefail` and trap-based cleanup; the verify script always attempts a graceful `osascript` quit before falling back to `pkill -f`. The pkill fallback pattern is narrow (`doc2md.app/Contents/MacOS/doc2md`) to avoid collateral damage.
- Local build is unsigned/ad-hoc signed. README documents this; signing and notarization are Phase 5.
- Codex sandbox used `danger-full-access` per persisted user preference; no secret paths or credentials touched.

### Decisions Made
- Do not persist security-scoped bookmarks or last-opened folder across launches in Phase 3; defer recent files/bookmarks/UserDefaults scope to Phase 4.
- Put visible error UX in the desktop shell status/toast region, with conflict UI directly under the toolbar.
- Use React save in-flight state as the fast double-trigger guard rather than adding a separate debounce.
- Use deprecated `NSOpenPanel/NSSavePanel.allowedFileTypes` intentionally because it gives exact `.md`, `.markdown`, and `.txt` extension filtering; `allowedContentTypes`/UTType widened `.txt` too far.
- Keep `load succeeded` log token as a weaker precondition and make `app ready: true` the launch-smoke pass condition.

### Security Notes
- Atomic writes use a sibling temp file and `FileManager.replaceItemAt`.
- Permission-denied paths map to `permission-needed` instead of generic errors.
- The native API allowlist is now positive and named; legitimate Phase 3 APIs are justified rather than silently accepted.
- Security-scoped access is balanced for selected URLs and not persisted across relaunch.

### Open Questions
- Phase 4 should decide bookmark persistence/recent files and whether last-opened folder moves from in-memory to UserDefaults.
- The intentional `allowedFileTypes` deprecation warning is a tradeoff for exact filtering; revisit only if Apple offers an exact modern alternative.
