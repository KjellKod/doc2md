### Security Notes
- Forbidden-API grep targets `apps/macos/doc2md/ShellBridge.swift` only. `AppScheme.swift` legitimately uses `FileManager` for the `doc2md://` scheme handler; keeping the grep scope narrow preserves that. Pattern: `\b(FileManager|NSOpenPanel|NSSavePanel|FileHandle|replaceItem|replacingItem|startAccessingSecurityScopedResource|stopAccessingSecurityScopedResource|securityScopedResource)\b|\.write\(to:`.
- Scripts use `set -euo pipefail` and trap-based cleanup; the verify script always attempts a graceful `osascript` quit before falling back to `pkill -f`. The pkill fallback pattern is narrow (`doc2md.app/Contents/MacOS/doc2md`) to avoid collateral damage.
- Local build is unsigned/ad-hoc signed. README documents this; signing and notarization are Phase 5.
- Codex sandbox used `danger-full-access` per persisted user preference; no secret paths or credentials touched.

### Open Questions
- Does the launch smoke need to handle the case where no TCC/Accessibility permission has been granted to the terminal running the script? Current behavior: AppleScript fails with a classifiable error and the script exits non-zero with a clear message. Documented in README; not a blocker.
- Phase 3 will introduce filesystem I/O in `ShellBridge.swift`. When that lands, the forbidden-API grep needs to be narrowed (or removed and replaced with a different guard). Record that when Phase 3 starts; do not preemptively weaken the tripwire.
- The launch smoke does not pre-kill a stale `doc2md` instance before `open -na`. Reviewer A's informational observation. Defensible because `-n` forces a new instance and the trap handles teardown. If flakiness surfaces, add a one-line pre-kill.
