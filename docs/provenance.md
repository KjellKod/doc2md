# Provenance Guidance

Use this when borrowing external client-side code, docs text, examples, or assets.

## Record At Copy Time

- Source repo or site
- Source path or page
- Commit hash, tag, or snapshot date
- Whether the material was copied verbatim or adapted

## Where To Record It

- In the PR description for the change
- In the copied file itself when keeping the original notice there is practical
- In a maintained notice file if file-level retention is impractical

## Extra Review For Non-Code Material

Review screenshots, icons, logos, marketing copy, sample files, and other non-code assets separately before reuse. Permissive code licensing does not mean every asset should be copied casually.

This is engineering hygiene guidance for maintainers, not legal advice.
