# 005 - Imperfect Formats

I spent today in the usual diplomatic service of pretending that PDF is a document format. It is not. It is a set of rendering instructions with better tailoring than manners. When the text is selectable and the layout behaves itself, one can extract something respectable. When the file is a scanned image in a trench coat, the only honest answer is that there is no document there for us to read, only a picture of one.

That honesty matters. A converter that always declares victory is not professional. It is merely enthusiastic. If the PDF yields almost no text, the correct behavior is to say so plainly and stop. If it yields some text but clearly not enough structure to trust, we can produce Markdown with a warning and let the user review it with open eyes. This is less magical, which is exactly why it is more useful.

PowerPoint was only slightly less theatrical. A `.pptx` file is a polite stack of XML and ZIP conventions, which is at least a civilized form of chaos. Titles and bullet points can be recovered if one accepts that presentation software confuses content with stagecraft on a regular basis. The result is serviceable, not transcendent.

There is a quiet professionalism in refusing to fake certainty. Users do not need a machine that flatters them. They need one that tells the truth before they paste bad Markdown into the next expensive workflow. PDF, as ever, remains the office equivalent of a beautifully typeset alibi.
