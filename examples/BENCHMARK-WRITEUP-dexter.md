# The One Second Witness

The updated 10 run dataset tightens the story. `doc2md` finished the mixed PDF, DOCX, XLSX, and PPTX batch in 0 to 1 seconds on every run. At this benchmark's one second resolution, that means a 1 second median and a 0.9 second average. The same extraction task took Claude 125 seconds median on the raw path and Codex 189 seconds. When `doc2md` handled format conversion first, Claude dropped to 69 seconds and Codex to 103.5. That is a 44.8 percent median reduction for Claude and a 45.2 percent reduction for Codex. The old 179 millisecond claim belonged to an earlier, narrower snapshot. The full dataset says something simpler and harder to argue with, the converter is effectively instantaneous next to the agents.

The important point is still not that the models are weak. They are not. They can read office documents, sometimes well enough to make people complacent. The problem is that they spend expensive reasoning time doing clerical extraction, probing binary files, trying fallback tools, and reconstructing text structures that should have been normalized before the model ever enters the room. Intelligence is being billed for janitorial work. Finance departments tend to notice eventually.

The Codex raw path is where the numbers start staring back. Same files, same task, same machine, yet run 3 took 274 seconds and run 8 took 145. That is a 129 second spread, almost another full fast run hiding inside the gap. The slowest raw run was 1.89 times the fastest. After `doc2md`, Codex stayed between 97 and 108 seconds, an 11 second spread, with the slowest run only 1.11 times the fastest. That is not a cosmetic improvement. That is the difference between a system that executes a procedure and a system that goes wandering through the walls looking for a light switch.

Claude shows the same pattern, just with less menace. Raw extraction ranged from 103 to 142 seconds. With `doc2md`, it tightened to 63 through 73. The medians matter, but the variance is what makes the operational case. Pipelines, SLAs, and cost estimates do not break because the average was disappointing. They break because the tail decided to improvise.

The most unsettling number may be the quietest one. The `doc2md` only column never left the 0 to 1 second band across all 10 runs. One zero, nine ones, no theatrics. At this level of timing resolution it behaves less like a model adjunct and more like infrastructure, which is exactly what a preprocessing step should be. The converter is the only participant here that never seems tempted to have a personality.

There is a broader lesson in that. The benchmark did not merely compare products. It exposed where uncertainty actually lives. The instability is not in turning documents into markdown. The instability is in letting an agent improvise its own extraction path from binary formats, tool by tool, guess by guess. `doc2md` does not make the models smarter. It removes a class of avoidable uncertainty before they start. That is a colder claim than the earlier writeup made, and a more useful one. Systems usually fail in the parts people thought were beneath discussion.
